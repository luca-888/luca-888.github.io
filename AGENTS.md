# AGENTS.md

这个仓库是一个个人静态博客，不是复杂应用。后续代理在这里工作时，应优先按“内容维护 + 少量样式调整”的方式处理，避免引入不必要的开发流程。

## 项目定位

- 这是一个 Astro 静态博客。
- 主要工作通常是新增或修改文章、调整页面样式、修正少量布局问题。
- 不要为了简单改动创建复杂架构、设计文档、长计划、子代理拆分或重型流程。

## 常用命令

- 测试：`npm test`
- 构建验证：`./node_modules/.bin/astro build`

如果 `npm run build` 因本地环境没有解析 `node_modules/.bin` 而失败，可以直接使用上面的 Astro CLI 命令。

## 工作原则

- 小改动直接做，保持修改范围清楚。
- 内容改动优先编辑 `src/content/blog/` 下的 Markdown 文件。
- 样式改动优先编辑 `src/styles/global.css`。
- 布局改动优先编辑 `src/layouts/BaseLayout.astro` 和 `src/pages/`。
- 不要随意引入新依赖；只有明确需要时才修改 `package.json`。
- 不要重写无关文件，也不要回滚用户已有改动。

## 清理规则

- 不要提交或保留生成物：`dist/`、`.astro/`。
- 不要提交依赖目录：`node_modules/`。
- 可以删除确认无引用的旧图片、空目录和过期文章文件。
- 删除资源前先用 `rg` 检查引用，避免删掉仍在文章或页面中使用的文件。

## 验证要求

- 纯文章修改：检查 Markdown 内容和资源引用即可。
- 样式或布局修改：优先运行 `./node_modules/.bin/astro build`。
- 测试相关修改：运行 `npm test`。
- 如果验证失败，直接说明失败原因和已确认的范围，不要假装通过。
